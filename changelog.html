Version 1.3.3

Bug fixes

The -s flag now correctly indexes a specific element if the returned result is an array.

When supplying invalid JSON the JGrep library will no longer exit.

When no matches are found the JGrep executable will now display an empty array instead of nothing.

Version 1.3.1

Features:

The -n flag has been added to allow for continuous input on the CLI.


       tail -f yourfile.json | jgrep -n "foo=bar"

The JGrep module now contains a JGrep class that can be used to set constant expressions and values.


    require 'jgrep.rb'

    a = JGrep::JGrep.new

    a.expression = "foo=bar"
    puts a.match_value('{"foo":"bar"}')

    a.value = {"foo" => "bar"}
    puts a.match_expression("foo=bar")

Version 1.3.0.1

Features:

The -q flag has been added which now allows jgrep to not write to stdout and instead exit with status code 0 if a match is found.

Version 1.3.0

Features:

The -s flag now supports wild cards on keys: 


    {
     "status": "OK",
     "data": {
       "baz.bar.com": {
         "classes": [
           "base"
         ],
         "roles": [
           "baserole"
         ],
         "parameters": {
           "time_server": "time.stanford.edu"
         }
       },

       "foo.bar.com": {
         "classes": [
           "base"
         ],
         "roles": [
           "baserole"
         ],
         "parameters": {
           "time_server": "time.mit.edu"
         }
       }
     }
    }

    $ jgrep -s data.*.parameters.time_server

    time.mit.edu
    time.standford.edu

Regular expressions can now be used in matcher expressions: 


    $ jgrep data.foo.bar.com.parameters.time_server=/time/
    [
        {
            "data": {
                "foo.bar.com": {
                    "parameters": {
                        "time_server": "time.mit.edu"
                    },
    ...

'And' and 'or' can now be replaced with '&&' and '||': 


    $ jgrep "data.foo.bar.com.parameters.time_server=/time/ && status=OK"
    [
        {
            "data": {
                "foo.bar.com": {
                    "parameters": {
                        "time_server": "time.mit.edu"
                    },
    ...

    $ jgrep "data.foo.bar.com.parameters.time_server=/time/ || status=NOTOK"
    [
        {
            "data": {
                "foo.bar.com": {
                    "parameters": {
                        "time_server": "time.mit.edu"
                    },
    ...

Bug Fixes


    - JGrep now correctly identifies dot seperated keys as keys.
    - JGrep is now correctly dependant on the JSON gem.
